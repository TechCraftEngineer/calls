# Реализация DateRangeFilter

## Задача 8.1

Реализован Client Component `DateRangeFilter` для выбора диапазона дат.

## Что реализовано

### ✅ Основной функционал

1. **Календарный компонент**
   - Использует `shadcn/ui Calendar` компонент
   - Режим выбора диапазона (`mode="range"`)
   - Отображение 2 месяцев одновременно
   - Блокировка будущих дат

2. **Быстрые фильтры**
   - Сегодня
   - Вчера
   - Последние 7 дней
   - Последние 30 дней
   - Текущий месяц
   - Интеграция с `getQuickFilterDates` из `date-range-utils.ts`

3. **Валидация периода**
   - Максимум 90 дней
   - Автоматический расчет количества дней
   - Предупреждение при превышении лимита
   - Блокировка применения невалидного периода

4. **UI/UX**
   - Popover для отображения календаря
   - Кнопка с иконкой календаря
   - Отображение выбранного периода в формате DD.MM.YYYY
   - Предупреждение с красным фоном при ошибке валидации

### ✅ Accessibility

1. **ARIA атрибуты**
   - `aria-label` на кнопке выбора периода
   - `aria-expanded` для состояния popover
   - `aria-label` на кнопках быстрых фильтров
   - `aria-label` на календаре
   - `role="alert"` и `aria-live="polite"` на предупреждении

2. **Keyboard Navigation**
   - Tab для навигации между элементами
   - Enter для выбора
   - Escape для закрытия popover
   - Стрелки для навигации по календарю (встроено в Calendar)

3. **Адаптивность**
   - Responsive дизайн (w-full на мобильных, md:w-[300px] на десктопе)
   - Минимальный размер кликабельных элементов соблюден
   - Grid layout для быстрых фильтров (2 колонки)

## Технические детали

### Управление состоянием

Компонент использует:
- `tempRange` - временное состояние для выбора в календаре
- `warning` - состояние для отображения предупреждений
- `open` - состояние открытия/закрытия popover

### Синхронизация с внешним состоянием

`useEffect` синхронизирует внутреннее состояние `tempRange` с внешними props `startDate` и `endDate`.

### Валидация

Валидация происходит при выборе обеих дат в календаре:
1. Рассчитывается количество дней между датами
2. Если > 90 дней - показывается предупреждение
3. Если <= 90 дней - вызывается `onChange` и закрывается popover

### Форматирование дат

- **Для отображения**: `DD.MM.YYYY` (русская локаль)
- **Для API**: `YYYY-MM-DD` (ISO формат)
