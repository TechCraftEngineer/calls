# TrendChart Component - Testing Guide

## Ручное тестирование

### Тест 1: Рендеринг с данными

**Шаги:**
1. Открыть страницу с компонентом TrendChart
2. Передать массив данных с несколькими днями

**Ожидаемый результат:**
- График отображается с двумя линиями
- Целевая линия (пунктирная, серая)
- Фактическая линия (сплошная, основной цвет)
- Точки на фактической линии окрашены в зависимости от процента выполнения

### Тест 2: Цветовое кодирование точек

**Шаги:**
1. Подготовить данные с разными процентами выполнения:
   - День 1: 120% (зеленый)
   - День 2: 85% (желтый)
   - День 3: 70% (красный)
2. Отобразить график

**Ожидаемый результат:**
- Точка дня 1 зеленая
- Точка дня 2 желтая
- Точка дня 3 красная

### Тест 3: Tooltip при наведении

**Шаги:**
1. Навести курсор на точку графика
2. Проверить содержимое tooltip

**Ожидаемый результат:**
- Tooltip отображается
- Показывает дату в формате "01 января 2024"
- Показывает фактическое время
- Показывает целевое время
- Показывает процент выполнения с цветовым кодированием

### Тест 4: Loading состояние

**Шаги:**
1. Передать `loading={true}`
2. Проверить отображение

**Ожидаемый результат:**
- Отображается skeleton loader
- Skeleton имеет высоту 400px

### Тест 5: Пустые данные

**Шаги:**
1. Передать пустой массив `data={[]}`
2. Передать `loading={false}`

**Ожидаемый результат:**
- Компонент не рендерится (возвращает null)
- Нет ошибок в консоли

### Тест 6: Responsive поведение

**Шаги:**
1. Открыть график на разных размерах экрана:
   - Мобильный (320px)
   - Планшет (768px)
   - Десктоп (1920px)

**Ожидаемый результат:**
- График адаптируется под ширину контейнера
- Все элементы читаемы на всех размерах
- Нет горизонтальной прокрутки

### Тест 7: Accessibility

**Шаги:**
1. Проверить наличие aria-labels на элементах графика
2. Попробовать навигацию с клавиатуры (Tab)
3. Проверить с помощью screen reader

**Ожидаемый результат:**
- Все элементы имеют aria-labels
- Tooltip доступен при фокусе
- Screen reader корректно читает информацию

## Проверка требований

### Requirement 10.1: Линейный график времени разговоров
✅ Реализовано: Компонент использует LineChart из Recharts

### Requirement 10.2: Целевая линия
✅ Реализовано: Пунктирная серая линия для целевых значений

### Requirement 10.3: Цветовое кодирование
✅ Реализовано: Точки окрашиваются в зеленый/желтый/красный

### Requirement 10.4: Tooltip с детальной информацией
✅ Реализовано: CustomTooltip показывает дату, время, процент

## Известные ограничения

1. Компонент не поддерживает виртуализацию для очень больших наборов данных (>90 дней)
2. Высота графика фиксирована (400px)
3. Формат даты на оси X фиксирован (DD.MM)

## Рекомендации по улучшению

1. Добавить возможность настройки высоты графика
2. Добавить zoom и pan для больших наборов данных
3. Добавить экспорт графика в изображение
4. Добавить анимацию при первом рендере
