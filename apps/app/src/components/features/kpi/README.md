# KPI Components

Компоненты для работы с KPI сотрудников.

## DateRangeFilter

Компонент для выбора диапазона дат с поддержкой быстрых фильтров и валидации.

### Возможности

- ✅ Календарный выбор периода (2 месяца одновременно)
- ✅ Быстрые фильтры (Сегодня, Вчера, Последние 7/30 дней, Текущий месяц)
- ✅ Валидация периода (максимум 90 дней)
- ✅ Предупреждение при превышении лимита
- ✅ Keyboard navigation (Tab, Enter, Escape)
- ✅ Accessibility (aria-labels, aria-live)
- ✅ Адаптивный дизайн (мобильные и десктоп)

### Использование

```tsx
import { DateRangeFilter } from "@/components/features/kpi";

function MyComponent() {
  const [startDate, setStartDate] = useState("2024-01-01");
  const [endDate, setEndDate] = useState("2024-01-31");

  const handleChange = (newStartDate: string, newEndDate: string) => {
    setStartDate(newStartDate);
    setEndDate(newEndDate);
  };

  return (
    <DateRangeFilter
      startDate={startDate}
      endDate={endDate}
      onChange={handleChange}
    />
  );
}
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `startDate` | `string` | Начальная дата в формате YYYY-MM-DD |
| `endDate` | `string` | Конечная дата в формате YYYY-MM-DD |
| `onChange` | `(startDate: string, endDate: string) => void` | Callback при изменении периода |

### Валидация

Компонент автоматически проверяет:
- Период не может превышать 90 дней
- Конечная дата не может быть в будущем
- Даты должны быть в формате YYYY-MM-DD

При превышении лимита в 90 дней отображается предупреждение, и изменения не применяются.

### Accessibility

- Все интерактивные элементы имеют `aria-label`
- Предупреждения используют `aria-live="polite"`
- Полная поддержка клавиатурной навигации
- Минимальный размер кликабельных элементов 44x44px на мобильных

### Зависимости

- `@calls/ui/components/button`
- `@calls/ui/components/calendar`
- `@calls/ui/components/popover`
- `@/lib/date-range-utils`
- `lucide-react` (CalendarIcon)

### Требования

Реализует требования спецификации:
- 5.1: Календарный компонент для выбора периода
- 5.2: Загрузка данных за выбранный период
- 5.5: Быстрые фильтры
- 5.6: Предупреждение при превышении 90 дней
- 7.4: Keyboard navigation
- 7.5: Aria-labels для доступности

## ExportButton

Компонент для экспорта данных KPI в CSV файл с обработкой ошибок и loading состоянием.

### Возможности

- ✅ Экспорт данных в CSV формат (UTF-8 с BOM)
- ✅ Loading состояние с spinner
- ✅ Обработка ошибок (пустые данные, ошибка генерации)
- ✅ Toast уведомления (успех/ошибка)
- ✅ Скачивание файла через blob URL
- ✅ Автоматическая очистка URL после скачивания
- ✅ Accessibility (aria-label)
- ✅ Disabled состояние при отсутствии данных

### Использование

```tsx
import { ExportButton } from "@/components/features/kpi";
import type { DailyKpiRow } from "@calls/api/routers/statistics/get-kpi-daily";

function MyComponent() {
  const data: DailyKpiRow[] = [
    // ... ваши данные
  ];

  return (
    <ExportButton
      data={data}
      employeeName="Иван Иванов"
      startDate="2024-01-01"
      endDate="2024-01-31"
    />
  );
}
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `data` | `DailyKpiRow[]` | Массив данных KPI по дням для экспорта |
| `employeeName` | `string` | Имя сотрудника (используется в имени файла) |
| `startDate` | `string` | Начальная дата периода в формате YYYY-MM-DD |
| `endDate` | `string` | Конечная дата периода в формате YYYY-MM-DD |

### Формат CSV файла

Генерируемый CSV файл содержит:
- UTF-8 BOM для корректного отображения кириллицы
- Заголовок с 11 колонками
- Точка с запятой (`;`) как разделитель
- Имя файла: `kpi-daily-{employeeName}-{startDate}-{endDate}.csv`

Пример структуры:
```csv
Дата;Сотрудник;Email;Входящие;Исходящие;Пропущенные;Всего звонков;Время разговоров (мин);Цель (мин);Выполнение (%);Бонус (₽)
2024-01-15;Иван Иванов;ivan@example.com;25;18;3;46;180;200;90;4500
```

### Обработка ошибок

Компонент обрабатывает следующие ошибки:
- **Пустые данные**: Показывает toast "Нет данных для экспорта"
- **Ошибка генерации**: Показывает toast с описанием ошибки
- **Любые другие ошибки**: Логирует в консоль и показывает общее сообщение об ошибке

### Состояния

- **Normal**: Кнопка активна, показывает иконку Download
- **Loading**: Кнопка disabled, показывает spinner и текст "Экспорт..."
- **Disabled**: Кнопка disabled при отсутствии данных (`data.length === 0`)

### Accessibility

- Кнопка имеет `aria-label="Экспортировать данные в CSV"`
- Disabled состояние корректно обрабатывается
- Минимальный размер кликабельного элемента соответствует стандартам

### Зависимости

- `@calls/ui/components/button`
- `@/lib/csv-export` (generateCSV, generateCSVFileName)
- `@/components/ui/toast` (useToast)
- `lucide-react` (Download, Loader2)

### Требования

Реализует требования спецификации:
- 6.1: Кнопка "Экспорт в CSV"
- 6.2: Генерация CSV файла с данными
- 6.3: Корректная структура CSV (11 колонок)
- 6.4: UTF-8 с BOM кодировка
- 6.5: Точка с запятой как разделитель
- 6.6: Формат имени файла
- 7.5: Aria-labels для доступности

## DailyStatsTable

Таблица с детализированной статистикой KPI по дням с сортировкой, пагинацией и цветовым кодированием.

### Возможности

- ✅ TanStack Table для управления таблицей
- ✅ 9 колонок: Дата, Входящие, Исходящие, Пропущенные, Всего, Время разговоров, Цель, Выполнение %, Бонус
- ✅ Сортировка по всем колонкам (ascending/descending)
- ✅ Пагинация (30 дней на страницу)
- ✅ Цветовое кодирование строк по проценту выполнения:
  - Зеленый: >= 100%
  - Желтый: >= 80% и < 100%
  - Красный: < 80%
- ✅ Строка с накопительными итогами (footer)
- ✅ Skeleton loading состояние
- ✅ Empty state с подсказкой
- ✅ Интегрированная кнопка экспорта
- ✅ Адаптивный дизайн
- ✅ Accessibility (aria-labels для сортировки)

### Использование

```tsx
import { DailyStatsTable } from "@/components/features/kpi";
import type { DailyKpiRow } from "@calls/api/routers/statistics/get-kpi-daily";

function MyComponent() {
  const { data, loading } = useKpiData(employeeId, startDate, endDate);

  return (
    <DailyStatsTable
      data={data}
      loading={loading}
      employeeName="Иван Иванов"
      startDate="2024-01-01"
      endDate="2024-01-31"
    />
  );
}
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `data` | `DailyKpiRow[]` | Массив данных KPI по дням |
| `loading` | `boolean` | Состояние загрузки данных |
| `employeeName` | `string` | Имя сотрудника (отображается в заголовке) |
| `startDate` | `string` | Начальная дата периода в формате YYYY-MM-DD |
| `endDate` | `string` | Конечная дата периода в формате YYYY-MM-DD |

### Структура таблицы

#### Колонки

1. **Дата** - Дата в формате DD.MM.YYYY
2. **Входящие** - Количество входящих звонков
3. **Исходящие** - Количество исходящих звонков
4. **Пропущенные** - Количество пропущенных звонков
5. **Всего** - Общее количество звонков
6. **Время разговоров** - Фактическое время в минутах
7. **Цель** - Целевое время в минутах
8. **Выполнение %** - Процент выполнения цели
9. **Бонус** - Рассчитанный бонус в рублях

#### Footer (Итоги)

Строка с накопительными итогами содержит:
- Сумму всех входящих, исходящих, пропущенных звонков
- Общее количество звонков
- Суммарное время разговоров и целевое время
- Общий процент выполнения
- Суммарный бонус

### Цветовое кодирование

Каждая строка таблицы окрашивается в зависимости от процента выполнения:

```tsx
// Зеленый (выполнение >= 100%)
bg-green-50 hover:bg-green-100 dark:bg-green-950/20

// Желтый (выполнение >= 80% и < 100%)
bg-yellow-50 hover:bg-yellow-100 dark:bg-yellow-950/20

// Красный (выполнение < 80%)
bg-red-50 hover:bg-red-100 dark:bg-red-950/20
```

### Сортировка

Все колонки поддерживают сортировку:
- Клик по заголовку колонки переключает сортировку (asc → desc → none)
- Визуальные индикаторы (стрелки вверх/вниз)
- Aria-labels для screen readers

### Пагинация

- Автоматически активируется при > 30 днях
- Показывает текущую страницу и общее количество
- Кнопки "Назад" и "Вперед" с иконками
- Информация о показанных записях

### Состояния

#### Loading

Показывает 10 skeleton строк с анимацией:

```tsx
<Skeleton className="h-12 w-full" />
```

#### Empty

Показывает карточку с сообщением:
- "Нет данных для отображения"
- "Попробуйте выбрать другой период"

#### Normal

Отображает таблицу с данными, пагинацией и кнопкой экспорта.

### Accessibility

- **Кнопки сортировки** имеют `aria-label` с описанием действия
- **Кнопки пагинации** имеют `aria-label`
- **Таблица** использует семантические HTML элементы (`<table>`, `<thead>`, `<tbody>`, `<tfoot>`)
- **Keyboard navigation** поддерживается через TanStack Table
