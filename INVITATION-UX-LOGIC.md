# 🎯 UX логика формы приглашения - исправлена

## ✅ Новая логика работы формы

### Сценарии поведения формы:

#### 1. **Пользователь авторизован + email совпадает**
- **Проверяем наличие пароля** через новый эндпоинт `checkUserPassword`
- **Если пароль есть:** показываем кнопку "Присоединиться к рабочему пространству"
- **Если пароля нет:** показываем форму создания пароля

#### 2. **Пользователь авторизован + email НЕ совпадает**
- Показываем сообщение "Несоответствие email"
- Предлагаем выйти или войти как другой пользователь

#### 3. **Пользователь НЕ авторизован**
- Показываем форму регистрации для нового пользователя
- Создаем аккаунт + пароль + присоединение к workspace

## 🔧 Технические улучшения

### 1. Новый API эндпоинт
```typescript
// check-user-password.ts
export const checkUserPassword = publicProcedure
  .input(z.object({ email: z.string().email() }))
  .handler(async ({ input, context }) => {
    // Проверяем наличие credential аккаунта у пользователя
    const accounts = await internalAdapter.findAccounts(user.id);
    const hasCredential = accounts.some((a) => a.providerId === "credential");
    
    return { hasPassword: hasCredential, exists: true };
  });
```

### 2. Обновленный фронтенд

## 🔄 Процесс работы

1. **Пользователь переходит по ссылке приглашения**
2. **Фронтенд проверяет авторизацию**
3. **Если авторизован:**
   - Проверяет email совпадение
   - Если совпадает → проверяет наличие пароля
   - Показывает нужный интерфейс
4. **Если не авторизован:**
   - Показывает форму регистрации
   - Создает пользователя + пароль + аккаунт
