# Архитектура запуска Audio Enhancer

## ✅ Правильная архитектура (уже реализована)

### Файлы и их назначение:

1. **`main.py`** - Только определение FastAPI приложения
   - Создает FastAPI app
   - Определяет эндпоинты
   - **НЕ запускает сервер**

2. **`Dockerfile`** - Запускает сервер
   ```dockerfile
   CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "7860", "--workers", "1"]
   ```

3. **`docker-compose.yml`** - Конфигурация контейнера
   - Порт 7860:7860
   - Переменные окружения
   - Health checks

### Способы запуска:

#### 1. Через Docker (только способ)
```bash
cd services/audio-enhancer
docker-compose up --build
```

#### 2. Через Docker напрямую
```bash
cd services/audio-enhancer
docker build -t audio-enhancer .
docker run -p 7860:7860 audio-enhancer
```

### ❌ Что НЕ нужно делать:

- **НЕ добавлять** `if __name__ == "__main__":` в main.py
- **НЕ вызывать** `uvicorn.run()` в main.py  
- **НЕ запускать** сервер автоматически при импорте

### 📋 Почему такая архитектура правильная:

1. **Разделение ответственности** - код для запуска отдельно от логики
2. **Docker-совместимость** - контейнер управляет запуском
3. **Гибкость** - можно запускать разными способами
4. **Тестирование** - легко импортировать без запуска сервера
5. **Production-ready** - соответствует лучшим практикам

### 🔧 Ports:
- **7860** - основной Docker порт
- Только Docker-контейнер использует этот порт

### 📝 Результат:
Сервис готов к запуску через Docker с правильной архитектурой микросервиса.
