# Руководство по test_diarization.sh

## Что делает скрипт

Скрипт выполняет полное тестирование speaker diarization с детальной диагностикой:

1. **Диагностика эмбеддингов** - проверяет качество speaker embeddings
2. **Полная транскрипция** - выполняет распознавание с диаризацией
3. **Анализ результатов** - даёт рекомендации по улучшению

## Использование

### Базовое использование
```bash
./test_diarization.sh audio.mp3
```

### С указанием API URL
```bash
./test_diarization.sh audio.mp3 https://your-api.com
```

### С прокси
```bash
./test_diarization.sh audio.mp3 https://your-api.com 127.0.0.1:2080
```

## Что проверяет скрипт

### 1. Качество аудио
- ✅ Sample rate (должен быть >= 16000 Hz)
- ✅ Автоматический апсемплинг
- ✅ Рекомендации по улучшению

### 2. Статистика эмбеддингов
- ✅ Pyannote загружена (должно быть true)
- ✅ Remote URL настроен
- ✅ Средняя норма эмбеддингов
- ✅ **Среднее косинусное расстояние** (ключевая метрика!)

### 3. Результаты диаризации
- ✅ Количество сегментов
- ✅ Количество уникальных спикеров
- ✅ Timeline по спикерам
- ✅ Время обработки

## Интерпретация результатов

### Avg Cosine Distance (ключевая метрика)

```
< 0.01   ❌ ПРОБЛЕМА - эмбеддинги идентичны
         → Проверьте размерность (должно быть 512, не 192)
         → Перезапустите speaker-embeddings сервис

0.01-0.20 ⚠️  Очень похожие голоса
         → Возможно один спикер
         → Или очень похожие голоса (два мужчины/две женщины)

0.20-0.40 ✅ Хорошее различие
         → 2+ спикера корректно различаются
         → Нормальная работа системы

> 0.40    ✅ Отличное различие
         → Явно разные голоса (мужской + женский)
         → Идеальные условия для диаризации
```

### Количество спикеров

**Если обнаружен только 1 спикер (SPEAKER_01):**

1. **Проверьте avg_cosine_distance:**
   - Если < 0.01 → проблема с эмбеддингами
   - Если > 0.20 → в аудио действительно один человек

2. **Проверьте pyannote_loaded:**
   - Должно быть `true`
   - Если `false` → установите HF_TOKEN

3. **Проверьте качество аудио:**
   - Sample rate >= 16000 Hz
   - Или auto_resample_enabled=true

## Настройка порога кластеризации

Если спикеры не различаются, но avg_distance хороший:

```bash
# Для похожих голосов (distance 0.20-0.30)
export CLUSTERING_BASE_THRESHOLD=0.25

# По умолчанию (distance 0.30-0.40)
export CLUSTERING_BASE_THRESHOLD=0.30

# Для очень разных голосов (distance > 0.40)
export CLUSTERING_BASE_THRESHOLD=0.35
```

После изменения:
```bash
docker-compose restart giga-am
./test_diarization.sh audio.mp3
```

## Требования

### Обязательно
- `curl` - для HTTP запросов
- Аудио файл в поддерживаемом формате

### Рекомендуется
- `jq` - для красивого форматирования JSON
  - Mac: `brew install jq`
  - Ubuntu: `sudo apt install jq`
  - Windows: скачать с https://stedolan.github.io/jq/

## Troubleshooting

### Ошибка: HTTP 500
```bash
# Проверьте логи сервиса
docker-compose logs giga-am

# Перезапустите сервис
docker-compose restart giga-am
```

### Ошибка: Connection refused
```bash
# Проверьте что сервис запущен
docker-compose ps

# Проверьте URL
curl https://your-api.com/api/health
```

### jq не установлен
```bash
# Mac
brew install jq

# Ubuntu/Debian
sudo apt install jq

# Windows (через Chocolatey)
choco install jq
```
