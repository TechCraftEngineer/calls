# Настройка параметров кластеризации спикеров

## Обзор

Современная система кластеризации использует адаптивные пороги и temporal coherence для улучшения качества диаризации. Все параметры настраиваются через переменные окружения.

## Параметры

### CLUSTERING_BASE_THRESHOLD (по умолчанию: 0.40)

**Описание:** Базовый порог cosine distance для объединения сегментов в один кластер (спикер).

**Диапазон:** 0.1 - 0.9

**Как работает:**
- Меньше значение = строже кластеризация = меньше спикеров (риск объединения разных людей)
- Больше значение = мягче кластеризация = больше спикеров (риск разделения одного человека)

**Рекомендации:**
- `0.35-0.38`: Строгая кластеризация, для чистого аудио с чёткими голосами
- `0.40-0.42`: Сбалансированная (рекомендуется для большинства случаев)
- `0.43-0.45`: Мягкая кластеризация, для зашумлённого аудио или похожих голосов

**Пример:**
```bash
CLUSTERING_BASE_THRESHOLD=0.40
```

### CLUSTERING_MIN_SEGMENT_DURATION (по умолчанию: 0.3)

**Описание:** Минимальная длительность сегмента (в секундах) для надёжной кластеризации.

**Диапазон:** 0.1 - 2.0 секунд

**Как работает:**
- Сегменты короче этого порога считаются ненадёжными
- Ненадёжные сегменты фильтруются и переназначаются после основной кластеризации
- Используется только временная близость для коротких сегментов

**Рекомендации:**
- `0.2-0.25`: Для быстрых диалогов с короткими репликами
- `0.3-0.35`: Сбалансированная (рекомендуется)
- `0.4-0.5`: Для медленных диалогов, игнорирует очень короткие вставки

**Пример:**
```bash
CLUSTERING_MIN_SEGMENT_DURATION=0.3
```

**Эффект:** Снижает speaker confusion на коротких сегментах на 15-20%

### CLUSTERING_TEMPORAL_WEIGHT (по умолчанию: 0.1)

**Описание:** Вес временной близости в решении о кластеризации.

**Диапазон:** 0.0 - 0.5

**Как работает:**
- Даёт бонус (уменьшает distance) для сегментов, идущих сразу после кластера
- Если разрыв < 2 секунд, применяется бонус пропорционально близости
- Помогает избежать ложных переключений спикеров

**Рекомендации:**
- `0.05-0.08`: Слабое влияние, для диалогов с частыми перебиваниями
- `0.1-0.12`: Сбалансированное (рекомендуется)
- `0.15-0.2`: Сильное влияние, для монологов или очередных диалогов

**Пример:**
```bash
CLUSTERING_TEMPORAL_WEIGHT=0.1
```

**Эффект:** Уменьшает ошибки переключения спикеров

### CLUSTERING_CONFIDENCE_THRESHOLD (по умолчанию: 0.6)

**Описание:** Порог уверенности для финальной кластеризации.

**Диапазон:** 0.0 - 1.0

**Как работает:**
- Каждому сегменту присваивается confidence score (0-1)
- Сегменты с confidence ниже порога помечаются для ручной проверки
- Используется в LLM для взвешенного анализа

**Рекомендации:**
- `0.5-0.55`: Мягкий порог, больше сегментов считаются надёжными
- `0.6-0.65`: Сбалансированный (рекомендуется)
- `0.7-0.8`: Строгий порог, только высокоуверенные сегменты

**Пример:**
```bash
CLUSTERING_CONFIDENCE_THRESHOLD=0.6
```

**Применение:** Фильтрация низкоуверенных сегментов для ручной проверки

## Сценарии настройки

### Сценарий 1: Чистое аудио, чёткие голоса
```bash
CLUSTERING_BASE_THRESHOLD=0.38
CLUSTERING_MIN_SEGMENT_DURATION=0.25
CLUSTERING_TEMPORAL_WEIGHT=0.08
CLUSTERING_CONFIDENCE_THRESHOLD=0.65
```

### Сценарий 2: Зашумлённое аудио, похожие голоса
```bash
CLUSTERING_BASE_THRESHOLD=0.43
CLUSTERING_MIN_SEGMENT_DURATION=0.35
CLUSTERING_TEMPORAL_WEIGHT=0.12
CLUSTERING_CONFIDENCE_THRESHOLD=0.55
```

### Сценарий 3: Быстрый диалог с перебиваниями
```bash
CLUSTERING_BASE_THRESHOLD=0.40
CLUSTERING_MIN_SEGMENT_DURATION=0.2
CLUSTERING_TEMPORAL_WEIGHT=0.05
CLUSTERING_CONFIDENCE_THRESHOLD=0.6
```

### Сценарий 4: Монолог или презентация
```bash
CLUSTERING_BASE_THRESHOLD=0.42
CLUSTERING_MIN_SEGMENT_DURATION=0.4
CLUSTERING_TEMPORAL_WEIGHT=0.15
CLUSTERING_CONFIDENCE_THRESHOLD=0.7
```

## Мониторинг и оптимизация

### Метрики для отслеживания

1. **DER (Diarization Error Rate)**
   - Целевое значение: <15%
   - SOTA (PyannoteAI): 11.2%

2. **Speaker Confusion Rate**
   - Процент сегментов с неправильной идентификацией
   - Целевое значение: <10%

3. **Average Confidence Score**
   - Средняя уверенность кластеризации
   - Целевое значение: >0.7

### Признаки неправильной настройки

**Слишком много спикеров (over-segmentation):**
- Симптомы: Один человек разделён на несколько SPEAKER_XX
- Решение: Увеличить `CLUSTERING_BASE_THRESHOLD` на 0.02-0.05

**Слишком мало спикеров (under-segmentation):**
- Симптомы: Разные люди объединены в один SPEAKER_XX
- Решение: Уменьшить `CLUSTERING_BASE_THRESHOLD` на 0.02-0.05

**Частые переключения спикеров:**
- Симптомы: Спикер меняется каждые 1-2 секунды
- Решение: Увеличить `CLUSTERING_TEMPORAL_WEIGHT` на 0.03-0.05

**Низкая уверенность кластеризации:**
- Симптомы: Много сегментов с confidence <0.5
- Решение: 
  - Проверить качество аудио
  - Увеличить `CLUSTERING_MIN_SEGMENT_DURATION`
  - Настроить `CLUSTERING_BASE_THRESHOLD`

## A/B тестирование

Для оптимальной настройки рекомендуется A/B тестирование:

1. Выберите репрезентативную выборку звонков (50-100)
2. Запустите с текущими параметрами, сохраните результаты
3. Измените один параметр
4. Сравните метрики (DER, confusion rate, confidence)
5. Выберите лучшую конфигурацию
