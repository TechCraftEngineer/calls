# Автоматический апсемплинг аудио

## Описание

Сервис автоматически улучшает качество аудио для лучшей диаризации спикеров.

Если входное аудио имеет sample rate < 16000 Hz, оно автоматически апсемплируется до 16000 Hz перед обработкой.

## Зачем это нужно

### Проблема с низким качеством:
- 8000 Hz: частотный диапазон 0-4000 Hz
- Теряется информация о тембре голоса
- Эмбеддинги становятся похожими
- Диаризация не работает (все спикеры → SPEAKER_01)

### С апсемплингом до 16000 Hz:
- Частотный диапазон: 0-8000 Hz
- Сохраняется больше характеристик голоса
- Эмбеддинги различаются
- Диаризация работает корректно (SPEAKER_01, SPEAKER_02, ...)

## Как это работает

1. Сервис проверяет sample rate входного аудио
2. Если < 16000 Hz → автоматический апсемплинг
3. Обработанное аудио используется для транскрипции и диаризации
4. Временный файл удаляется после обработки

## Логи

### Аудио достаточного качества:
```
[request-id] Аудио качество достаточное: 44100Hz (target: 16000Hz)
```

### Автоматический апсемплинг:
```
[request-id] Низкое качество аудио: 8000Hz. Автоматический апсемплинг до 16000Hz...
[request-id] Аудио успешно апсемплировано: 8000Hz → 16000Hz
```

## Настройки

### Включение/отключение

```bash
# Включено по умолчанию
export AUTO_RESAMPLE_ENABLED=true

# Отключить (не рекомендуется)
export AUTO_RESAMPLE_ENABLED=false
```

### Целевой sample rate

```bash
# По умолчанию: 16000 Hz
export TARGET_SAMPLE_RATE=16000

# Для ещё лучшего качества (медленнее)
export TARGET_SAMPLE_RATE=22050

# Для максимального качества (значительно медленнее)
export TARGET_SAMPLE_RATE=44100
```

## Производительность

### Влияние на скорость:

| Sample Rate | Размер файла | Время обработки |
|-------------|--------------|-----------------|
| 8000 Hz     | 1x           | +0.5-1s (ресемплинг) |
| 16000 Hz    | 2x           | 0s (без ресемплинга) |
| 44100 Hz    | 5.5x         | 0s (без ресемплинга) |

Апсемплинг добавляет 0.5-1 секунду на обработку, но значительно улучшает качество диаризации.

## Примеры

### До (без апсемплинга):
```json
{
  "audio_metadata": {
    "sample_rate": 8000
  },
  "segments": [
    {"speaker": "SPEAKER_01", "text": "..."},
    {"speaker": "SPEAKER_01", "text": "..."},
    {"speaker": "SPEAKER_01", "text": "..."}
  ]
}
```

### После (с апсемплингом):
```json
{
  "audio_metadata": {
    "sample_rate": 8000,
    "resampled_to": 16000
  },
  "segments": [
    {"speaker": "SPEAKER_01", "text": "..."},
    {"speaker": "SPEAKER_02", "text": "..."},
    {"speaker": "SPEAKER_01", "text": "..."}
  ]
}
```

## Рекомендации

### Для продакшена:
```bash
# Оставьте настройки по умолчанию
AUTO_RESAMPLE_ENABLED=true
TARGET_SAMPLE_RATE=16000
```

### Для высокого качества:
```bash
# Если производительность не критична
AUTO_RESAMPLE_ENABLED=true
TARGET_SAMPLE_RATE=22050
```

### Для отладки:
```bash
# Отключите, чтобы видеть проблемы с оригинальным аудио
AUTO_RESAMPLE_ENABLED=false
```

## Совместимость

Работает со всеми поддерживаемыми форматами:
- MP3, WAV, FLAC, M4A, AAC, OGG, WEBM

## Ограничения

- Апсемплинг не добавляет реальную информацию, только интерполирует
- Лучше отправлять изначально качественное аудио (16kHz+)
- Для критичных случаев рекомендуется предобработка на клиенте

## Диагностика

Проверьте, работает ли апсемплинг:

```bash
curl -X POST http://localhost:7860/api/debug-embeddings \
  -F "file=@low_quality_8khz.mp3" \
  | jq '.audio_quality'
```

Ожидаемый результат:
```json
{
  "sample_rate": 8000,
  "quality": "poor",
  "recommendation": "Увеличьте sample rate до 16000Hz",
  "auto_resampled": true,
  "resampled_to": 16000
}
```
