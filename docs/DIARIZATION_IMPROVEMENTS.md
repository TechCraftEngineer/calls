## 🔧 Рекомендуемые улучшения (по приоритету)

### Приоритет 1: Критические улучшения

#### 1.1 Adaptive Clustering Thresholds
**Проблема:** Фиксированный порог 0.40 не учитывает качество эмбеддингов и длительность сегментов

**Решение:** Реализовано в `clustering_service_v2.py`
- Динамический порог на основе длительности сегмента
- Оценка качества эмбеддинга (норма, разреженность)
- Короткие сегменты (<0.3с) требуют более строгого порога

**Ожидаемый эффект:** Снижение DER (Diarization Error Rate) на 2-3%

#### 1.2 Temporal Coherence
**Проблема:** Не учитывается временная близость сегментов

**Решение:** Реализовано в `clustering_service_v2.py`
- Бонус за временную близость (temporal bonus)
- Если сегмент идёт сразу после кластера, даём небольшой бонус
- Вес temporal bonus: 0.1 (настраиваемый)

**Ожидаемый эффект:** Меньше ошибок переключения спикеров

#### 1.3 Unreliable Segment Filtering
**Проблема:** Короткие сегменты (<0.3с) дают ненадёжные эмбеддинги

**Решение:** Реализовано в `clustering_service_v2.py`
- Фильтрация коротких сегментов перед основной кластеризацией
- Переназначение после кластеризации надёжных сегментов
- Использование только временной близости для ненадёжных

**Ожидаемый эффект:** Снижение speaker confusion на 15-20%

**Источник:** [Benchmarking Diarization Models (2024)](https://arxiv.org/html/2509.26177v1)
> "The primary cause of diarization errors stem from missed speech segments followed by speaker confusion"

### Приоритет 2: Важные улучшения

#### 2.1 Confidence Scoring
**Проблема:** Нет оценки уверенности кластеризации

**Решение:** Реализовано в `clustering_service_v2.py`
- Confidence score для каждого сегмента (0.0 - 1.0)
- На основе cosine distance и adaptive threshold
- Передача confidence в LLM для взвешенного анализа

**Применение:**
- Фильтрация низкоуверенных сегментов для ручной проверки

#### 2.3 Speaker Embedding Cache
**Проблема:** Повторное вычисление эмбеддингов для одних и тех же спикеров

**Решение:**
- Кэширование эмбеддингов по workspace + speaker

**Применение:**

### Приоритет 3: Оптимизации

#### 3.1 Batch Embedding Optimization
**Проблема:** Последовательная обработка сегментов

**Решение:**
- Батчевая обработка эмбеддингов (уже частично реализовано)
- Оптимизация размера батча для GPU/CPU

#### 3.2 Streaming Diarization
**Проблема:** Только offline обработка

**Решение:** Рассмотреть DIART или подобные для online diarization
- Полезно для real-time транскрипции
- Требует значительных изменений архитектуры

**Источник:** [DIART optimization (2024)](https://arxiv.org/html/2408.02341v1)

## 📊 Современные метрики качества

### Diarization Error Rate (DER)
Основная метрика качества диаризации:
```
DER = (FA + MISS + CONF) / TOTAL
```
- FA (False Alarm): ошибочно детектированная речь
- MISS: пропущенная речь
- CONF (Confusion): ошибка идентификации спикера

**Benchmark (2024):**
- PyannoteAI: 11.2% DER (SOTA)
- DiariZen: 13.3% DER (open-source)
- Наша цель: <15% DER

### Speaker Confusion Rate
Процент сегментов с неправильной идентификацией спикера

**Текущая проблема:** Высокий confusion в коротких сегментах

- Adaptive thresholds для разных длительностей сегментов

- Two-pass clustering для уменьшения ложных спикеров
- Filtering unreliable segments перед основной кластеризацией
