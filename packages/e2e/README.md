# @calls/e2e

Пакет для E2E тестирования с использованием Playwright.

## ⚡ Оптимизация производительности

**Тесты медленные?** См. [`QUICK_START_OPTIMIZATION.md`](./QUICK_START_OPTIMIZATION.md) для ускорения в 3-4 раза.

**Краткая сводка:**
- Применить `playwright.config.optimized.ts` → +30-40% скорости
- Добавить `data-testid` в компоненты → +50-100ms на селектор  
- Использовать оптимизированные helpers → +500-1000ms на тест

**Результат:** 3-5s → 1-2s на тест, 60-100s → 10-15s на 20 тестов

**Документация:**
- [`РЕЗЮМЕ.md`](./РЕЗЮМЕ.md) - краткая сводка на русском 🇷🇺
- [`SUMMARY.md`](./SUMMARY.md) - краткая сводка на английском 🇬🇧
- [`QUICK_START_OPTIMIZATION.md`](./QUICK_START_OPTIMIZATION.md) - пошаговая инструкция (3 шага, 45 минут)
- [`ACTION_CHECKLIST.md`](./ACTION_CHECKLIST.md) - детальный чеклист с галочками
- [`PERFORMANCE_OPTIMIZATION.md`](./PERFORMANCE_OPTIMIZATION.md) - детальное описание всех оптимизаций
- [`BEFORE_AFTER_COMPARISON.md`](./BEFORE_AFTER_COMPARISON.md) - визуальное сравнение до/после
- [`DATA_TESTID_CHECKLIST.md`](./DATA_TESTID_CHECKLIST.md) - список data-testid для компонентов

## Установка

```bash
# Установка зависимостей
bun install

# Установка браузеров Playwright
bun run install
```

## Запуск тестов

```bash
# Запуск всех тестов
bun run test

# Запуск тестов с UI
bun run test:ui

# Запуск только тестов аутентификации
bun run test:auth

# Запуск в режиме отладки
bun run test:debug

# Запуск с видимыми браузерами
bun run test:headed
```

## Структура тестов

```
tests/
├── auth/                    # Тесты аутентификации
│   ├── signin.spec.ts      # Тесты страницы входа
│   ├── signup.spec.ts      # Тесты страницы регистрации
│   ├── forgot-password.spec.ts # Тесты восстановления пароля
│   ├── auth-flow.spec.ts   # Тесты потока аутентификации
│   └── auth-with-fixtures.spec.ts # Тесты с фикстурами
├── fixtures/               # Фикстуры для тестов
│   └── auth.ts            # Фикстуры аутентификации
└── helpers/               # Вспомогательные функции
    └── auth-helpers.ts    # Помощники для тестов аутентификации
```

## Особенности тестов

### Тесты аутентификации

- **signin.spec.ts** - проверяет форму входа, валидацию, отправку данных
- **signup.spec.ts** - проверяет форму регистрации, валидацию полей
- **forgot-password.spec.ts** - проверяет восстановление пароля
- **auth-flow.spec.ts** - проверяет общий поток аутентификации, навигацию, доступность
- **auth-with-fixtures.spec.ts** - использует фикстуры для тестирования с разными пользователями

### Фикстуры

Фикстуры предоставляют тестовые данные:
- `validUser` - валидный пользователь для тестов
- `invalidUser` - невалидный пользователь для тестов ошибок
- `testUsers` - массив пользователей для массовых тестов

### Помощники

`AuthHelpers` класс содержит методы для:
- Заполнения форм аутентификации
- Мокирования API ответов
- Проверки элементов страниц
- Тестирования доступности

## Конфигурация

Конфигурация находится в `playwright.config.ts`:
- Поддержка множественных браузеров (Chrome, Firefox, Safari)
- Мобильное тестирование
- Автоматический запуск dev сервера
- Скриншоты при ошибках
- Трейсинг для отладки

## Переменные окружения

- `BASE_URL` - базовый URL приложения (по умолчанию: http://localhost:3000)
- `CI` - режим CI/CD (влияет на количество ретраев и воркеров)
