# Объединение последовательных сегментов одного спикера

## Описание

Функция `mergeConsecutiveSpeakerSegments` объединяет последовательные сегменты диаризации, принадлежащие одному и тому же спикеру, в единый интервал. Это улучшает читаемость транскриптов и делает диалоги более естественными.

## Проблема

До внедрения этой функции диаризация могла создавать множество коротких сегментов для одного спикера, что приводило к:
- Фрагментированным диалогам
- Сложности восприятия текста
- Избыточному количеству сегментов в базе данных

### Пример до оптимизации:
```text
SPEAKER_00: Привет
SPEAKER_00: как
SPEAKER_00: дела?
SPEAKER_01: Отлично
SPEAKER_01: спасибо
```

### Пример после оптимизации:
```text
SPEAKER_00: Привет как дела?
SPEAKER_01: Отлично спасибо
```

## Реализация

Функция применяется в двух ключевых точках пайплайна транскрибации:

1. **После асинхронной диаризированной транскрибации** (`async-transcription.ts`)
   - Объединяет сегменты с текстом, полученные от GigaAM API
   - Улучшает читаемость транскриптов

2. **После LLM мерджинга** (`merge-results.ts`)
   - Объединяет финальные сегменты после объединения результатов разных ASR
   - Финальная оптимизация перед сохранением в БД

**Важно:** Сегменты из speaker-embeddings диаризации НЕ объединяются перед передачей в GigaAM, чтобы сохранить детализацию временных меток для точной транскрипции.

## Алгоритм

1. Сортирует сегменты по времени начала
2. Пропускает пустые сегменты (без текста)
3. Объединяет последовательные сегменты одного спикера:
   - Расширяет временной интервал (`start` - `end`)
   - Конкатенирует текст через пробел
   - Усредняет `confidence` (если присутствует)
4. При смене спикера начинает новый сегмент

## Преимущества

- **Улучшенная читаемость**: Диалоги выглядят более естественно
- **Меньше данных**: Сокращение количества сегментов в БД
- **Лучшая производительность**: Меньше операций при обработке сегментов
- **Логирование**: Автоматически логирует процент сокращения сегментов

## Тестирование

Функция покрыта unit-тестами (`merge-consecutive-segments.test.ts`), которые проверяют:
- Объединение последовательных сегментов
- Обработку чередующихся спикеров
- Пропуск пустых сегментов
- Усреднение confidence
- Сортировку по времени
- Граничные случаи (пустой массив, один сегмент)

Запуск тестов:
```bash
bun test merge-consecutive-segments
```

## Использование

```typescript
import { mergeConsecutiveSpeakerSegments } from "./merge-consecutive-segments";

const segments = [
  { speaker: "SPEAKER_00", start: 0, end: 2, text: "Привет", confidence: 0.9 },
  { speaker: "SPEAKER_00", start: 2, end: 4, text: "как дела?", confidence: 0.8 },
  { speaker: "SPEAKER_01", start: 4, end: 6, text: "Отлично", confidence: 0.95 },
];

const merged = mergeConsecutiveSpeakerSegments(segments, "call-123");
// Результат:
// [
//   { speaker: "SPEAKER_00", start: 0, end: 4, text: "Привет как дела?", confidence: 0.85 },
//   { speaker: "SPEAKER_01", start: 4, end: 6, text: "Отлично", confidence: 0.95 }
// ]
```

## Логирование

При вызове с `callId` функция автоматически логирует статистику:
```text
Объединение последовательных сегментов одного спикера {
  callId: "call-123",
  originalCount: 10,
  mergedCount: 4,
  reductionPercent: 60
}
```
