# ASR Pipeline

Конвейер обработки аудио для транскрибации телефонных звонков.

## Этапы обработки

1. **Предобработка аудио** (`audio-preprocessing.ts`) ⭐ НОВОЕ
   - Автоматический fallback: Python ML → FFmpeg → без обработки
   - Python ML (если доступен): noisereduce, Silero VAD, FFT-обработка
   - FFmpeg (fallback): нормализация, усиление частот, базовое шумоподавление
   - Без обработки (если ничего не доступно): используется исходное аудио

2. **Multi-ASR распознавание** (параллельно)
   - AssemblyAI
   - Yandex SpeechKit
   - Hugging Face (несколько моделей)

3. **LLM объединение** (`merge-asr.ts`)
   - Объединяет результаты от разных ASR провайдеров
   - Выбирает лучшие фрагменты из каждого источника
   - Учитывает контекст при выборе между похожими словами

4. **Контекстная коррекция** (`context-correction.ts`) ⭐ НОВОЕ
   - Исправляет ошибки ASR с учетом контекста разговора
   - Примеры: "пирог" → "срок" (в контексте сроков поставки)
   - Использует информацию о компании для лучшего понимания терминологии
   - Консервативный подход: исправляет только явные ошибки

5. **LLM нормализация** (`normalize.ts`)
   - Исправление орфографии и пунктуации
   - Форматирование диалога
   - Удаление слов-паразитов

6. **Анализ и суммаризация** (`summarize.ts`)
   - Краткое содержание
   - Определение темы и типа звонка
   - Анализ тональности

## Использование

### Базовый пример (рекомендуется)

```typescript
import { runTranscriptionPipeline } from "@calls/jobs";

const result = await runTranscriptionPipeline(audioUrl, {
  companyContext: "Компания занимается поставками промышленного оборудования",
  // Предобработка включена по умолчанию
  // Автоматический fallback: Python ML → FFmpeg → без обработки
});
```

**Что происходит:**
1. Сначала пытается использовать Python ML сервис (если `AUDIO_ENHANCER_URL` задан)
2. Если Python недоступен → автоматически использует FFmpeg
3. Если FFmpeg недоступен → продолжает без предобработки

### Принудительное использование только FFmpeg

```typescript
const result = await runTranscriptionPipeline(audioUrl, {
  audioPreprocessing: {
    usePythonEnhancer: false, // Отключить Python, использовать только FFmpeg
    normalizeVolume: true,
    enhanceSpeech: true,
  },
});
```

### Отключение предобработки

```typescript
const result = await runTranscriptionPipeline(audioUrl, {
  skipAudioPreprocessing: true, // Полностью отключить предобработку
});
```

## Опции

- `skipAudioPreprocessing` - пропустить предобработку аудио (по умолчанию: false)
- `audioPreprocessing` - настройки предобработки:
  - `usePythonEnhancer` - использовать Python ML (по умолчанию: true, автоматический fallback на FFmpeg)
  - `normalizeVolume` - нормализация громкости (по умолчанию: true)
  - `enhanceSpeech` - усиление речевых частот (по умолчанию: true)
  - `noiseReduction` - шумоподавление (по умолчанию: false)
  - `removeSilence` - удаление пауз (по умолчанию: false)
- `skipContextCorrection` - пропустить контекстную коррекцию (по умолчанию: false)
- `skipNormalization` - пропустить нормализацию (по умолчанию: false)
- `companyContext` - информация о компании для улучшения распознавания
- `summaryPrompt` - кастомный промпт для суммаризации

## Предобработка аудио

Автоматический fallback обеспечивает максимальное качество при любой конфигурации.

### Приоритет обработки

1. **Python ML сервис** (если доступен)
   - ML-based шумоподавление (noisereduce)
   - Silero VAD для детекции речи
   - FFT-based усиление частот
   - +40% качество vs FFmpeg

2. **FFmpeg** (fallback, если Python недоступен)
   - Нормализация громкости (dynaudnorm)
   - Усиление речевых частот (equalizer)
   - Базовое шумоподавление (afftdn)
   - Быстро и надежно

3. **Без обработки** (если ничего не доступно)
   - Используется исходное аудио
   - Pipeline продолжает работу

### Настройка

**По умолчанию (рекомендуется):**
```typescript
// Автоматически пытается использовать Python ML
// Если недоступен → FFmpeg
// Если FFmpeg недоступен → без обработки
const result = await runTranscriptionPipeline(audioUrl);
```

**Только FFmpeg:**
```typescript
const result = await runTranscriptionPipeline(audioUrl, {
  audioPreprocessing: {
    usePythonEnhancer: false, // Принудительно FFmpeg
  },
});
```

**Без предобработки:**
```typescript
const result = await runTranscriptionPipeline(audioUrl, {
  skipAudioPreprocessing: true,
});
```

## Контекстная коррекция

Этап обработки, который исправляет типичные ошибки ASR с учетом контекста.

### Примеры исправлений

| Ошибка ASR | Контекст | Исправление |
|------------|----------|-------------|
| "пирог доставки" | Разговор о поставках | "срок доставки" |
| "кот активации" | Разговор о ПО | "код активации" |
| "банк топлива" | Разговор о технике | "бак топлива" |
| "мышка плюшевая" | Разговор об игрушках | "мишка плюшевая" |

### Принципы работы

1. Анализирует тему и контекст всего разговора
2. Находит слова, которые не подходят по смыслу
3. Подбирает фонетически похожие слова, подходящие по контексту
4. Использует информацию о компании (если предоставлена)
5. Консервативный подход: исправляет только явные ошибки

### Безопасность

- НЕ добавляет новые слова
- НЕ изменяет разговорный стиль
- НЕ исправляет сленг и диалектизмы
- Сохраняет имена собственные

## Рекомендации

### Для лучшего качества ASR

1. ✅ Включите предобработку аудио (особенно нормализацию громкости)
2. ✅ Предоставьте `companyContext` для лучшего понимания терминологии
3. ✅ Используйте контекстную коррекцию
4. ⚠️ Шумоподавление включайте только для очень зашумленных записей
5. ⚠️ Удаление пауз может удалить короткие слова

### Производительность

- Предобработка добавляет ~2-5 секунд на запись
- ASR провайдеры работают параллельно
- Контекстная коррекция добавляет ~1-2 секунды
- Общее время: ~10-30 секунд на минуту аудио
