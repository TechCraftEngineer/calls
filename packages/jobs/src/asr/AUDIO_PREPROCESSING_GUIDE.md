# Руководство по предобработке аудио для ASR

## Зачем нужна предобработка?

ASR системы чувствительны к качеству аудио. Предобработка может улучшить точность распознавания на 15-40% для записей с проблемами:

- 🔇 Тихие участки (слова произнесены тихо)
- 📢 Переменная громкость (то громко, то тихо)
- 🔊 Фоновый шум (офис, улица, техника)
- 📞 Плохое качество связи (телефонные звонки)

## Что делает каждый фильтр

### 1. Нормализация громкости ✅ РЕКОМЕНДУЕТСЯ

**Проблема:** Слово "срок" произнесено тихо → ASR распознает как "пирог"

**Решение:** `dynaudnorm` выравнивает громкость по всей записи

**Эффект:**
- ✅ Тихие слова распознаются правильно
- ✅ Стабильная точность на всей записи
- ✅ Работает для любых записей
- ⚠️ Добавляет ~1-2 секунды обработки

**Когда использовать:** ВСЕГДА (включено по умолчанию)

### 2. Усиление речевых частот ✅ РЕКОМЕНДУЕТСЯ

**Проблема:** Гул, шипение, высокочастотный шум мешают ASR

**Решение:** Фильтры highpass/lowpass/equalizer усиливают речь (300Hz-3400Hz)

**Эффект:**
- ✅ Четче речь на зашумленных записях
- ✅ Убирает гул и шипение
- ✅ Безопасно для любых записей
- ⚠️ Добавляет ~0.5 секунды обработки

**Когда использовать:** ВСЕГДА (включено по умолчанию)

### 3. Шумоподавление ⚠️ ОСТОРОЖНО

**Проблема:** Сильный фоновый шум (офис, улица, техника)

**Решение:** `afftdn` (FFT-based denoiser) убирает стационарный шум

**Эффект:**
- ✅ Убирает постоянный фоновый шум
- ❌ Может искажать речь (артефакты)
- ❌ Может удалить тихие согласные
- ⚠️ Добавляет ~2-3 секунды обработки

**Когда использовать:** 
- ✅ Очень зашумленные записи (стройка, завод)
- ❌ Обычные офисные записи (лучше без него)
- ❌ Записи с переменным шумом (не поможет)

### 4. Удаление пауз ⚠️ РЕДКО

**Проблема:** Длинные паузы замедляют обработку

**Решение:** `silenceremove` удаляет тишину >1 секунды

**Эффект:**
- ✅ Ускоряет обработку
- ❌ Может удалить короткие слова
- ❌ Может нарушить синхронизацию
- ⚠️ Добавляет ~1 секунду обработки

**Когда использовать:**
- ✅ Записи с очень длинными паузами
- ❌ Обычные разговоры (не нужно)

## Рекомендуемые настройки

### По умолчанию (автоматический fallback) ✅ РЕКОМЕНДУЕТСЯ

```typescript
// Ничего не указываем - система автоматически выберет лучший метод:
// 1. Python ML (если доступен)
// 2. FFmpeg (если Python недоступен)
// 3. Без обработки (если ничего не доступно)

audioPreprocessing: {
  usePythonEnhancer: true,  // По умолчанию (автоматический fallback)
  normalizeVolume: true,    // ✅ Всегда
  enhanceSpeech: true,      // ✅ Всегда
  noiseReduction: false,    // ❌ Осторожно
  removesilence: false,     // ❌ Редко нужно
}
```

### Принудительно только FFmpeg

```typescript
audioPreprocessing: {
  usePythonEnhancer: false, // Отключить Python, использовать только FFmpeg
  normalizeVolume: true,
  enhanceSpeech: true,
}
```

### Для очень плохих записей (с Python ML)

```typescript
audioPreprocessing: {
  usePythonEnhancer: true,  // Автоматический fallback на FFmpeg
  normalizeVolume: true,
  enhanceSpeech: true,
  noiseReduction: true,     // ⚠️ ML шумоподавление (если Python доступен)
  removesilence: false,
}
```

### Без предобработки (не рекомендуется)

```typescript
skipAudioPreprocessing: true
```

## Требования

### Автоматический fallback

Система автоматически выбирает лучший доступный метод обработки:

1. **Python ML сервис** (если `AUDIO_ENHANCER_URL` задан)
   - ML-based шумоподавление (noisereduce)
   - Silero VAD для детекции речи
   - FFT-based усиление частот
   - Лучшее качество (+40% vs FFmpeg)

2. **FFmpeg** (если Python недоступен)
   - Нормализация громкости (dynaudnorm)
   - Усиление речевых частот (equalizer)
   - Базовое шумоподавление (afftdn)
   - Быстро и надежно

3. **Без обработки** (если ничего не доступно)
   - Используется исходное аудио
   - Pipeline продолжает работу

### Установка FFmpeg (fallback)

**Ubuntu/Debian:**
```bash
sudo apt-get install ffmpeg
```

**macOS:**
```bash
brew install ffmpeg
```

**Windows:**
Скачайте с https://ffmpeg.org/download.html

**Проверка:**
```bash
ffmpeg -version
```

### Установка Python ML сервиса (рекомендуется)

См. `services/audio-enhancer/README.md` для инструкций по развертыванию.

## Производительность

| Фильтр | Время на 1 мин аудио | Улучшение точности |
|--------|---------------------|-------------------|
| Нормализация громкости | ~1-2 сек | +15-30% (тихие слова) |
| Усиление речевых частот | ~0.5 сек | +5-15% (шумные записи) |
| Шумоподавление | ~2-3 сек | +10-20% (очень шумные) |
| Удаление пауз | ~1 сек | 0% (только ускорение) |

**Итого:** ~2-5 секунд на минуту аудио (с настройками по умолчанию)

```typescript
import { runTranscriptionPipeline } from "@calls/jobs";

// Автоматический fallback: Python ML → FFmpeg → без обработки
const result = await runTranscriptionPipeline(audioUrl, {

  // usePythonEnhancer: true (по умолчанию)

    usePythonEnhancer: false, // Отключить Python, использовать только FFmpeg
    normalizeVolume: true,

  },

```

## Отладка

### Логи автоматического fallback

**Python ML успешно:**
```
[asr-audio-preprocessing] Попытка использовать Python ML сервис для обработки аудио
[asr-audio-preprocessing] Python ML обработка успешна
[asr-pipeline] Аудио предобработано
```

**Python недоступен → FFmpeg:**
```
[asr-audio-preprocessing] Попытка использовать Python ML сервис для обработки аудио
[asr-audio-preprocessing] Python сервис недоступен, используем FFmpeg fallback
[asr-audio-preprocessing] Используем FFmpeg для обработки аудио
[asr-audio-preprocessing] Предобработка аудио завершена
```

**FFmpeg недоступен → без обработки:**
```
[asr-audio-preprocessing] Python сервис недоступен, используем FFmpeg fallback
[asr-audio-preprocessing] FFmpeg не найден, пропускаем предобработку аудио
```

**Ошибка:**
```
[asr-audio-preprocessing] Ошибка предобработки аудио
```

В случае ошибки pipeline продолжит работу с исходным аудио.

## FAQ

**Q: Нужно ли всегда использовать предобработку?**  
A: Да, особенно нормализацию громкости. Она решает проблему с тихими словами.

**Q: Какой метод используется по умолчанию?**  
A: Автоматический fallback: сначала пытается Python ML, затем FFmpeg, затем без обработки.

**Q: Почему шумоподавление выключено по умолчанию?**  
A: Оно может искажать речь. Включайте только для очень зашумленных записей.

**Q: Сколько времени добавляет предобработка?**  
A: Python ML: ~5-8 секунд, FFmpeg: ~2-5 секунд на минуту аудио.

**Q: Что если Python сервис недоступен?**  
A: Автоматически используется FFmpeg. Если FFmpeg тоже недоступен, продолжает без обработки.

**Q: Нужно ли устанавливать Python сервис?**  
A: Нет, это опционально. FFmpeg fallback работает хорошо для большинства случаев.

**Q: Можно ли использовать свои FFmpeg фильтры?**  
A: Пока нет, но можно расширить `buildFFmpegFilters()` в `audio-preprocessing.ts`.

**Q: Влияет ли предобработка на качество голоса?**  
A: Минимально. Нормализация и усиление речевых частот безопасны. Шумоподавление может добавить артефакты.
